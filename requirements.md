📋 クラロワ 高度戦績分析ダッシュボード 要件定義書
1. プロジェクトの目的と概要
クラッシュ・ロワイヤルの公式APIから取得したバトルログをGoogle スプレッドシートに永続的に蓄積し、それをデータソースとして高度な統計・相性分析を行うStreamlitベースのローカルWebアプリケーションを構築する。

2. 前提環境・技術スタック（厳守事項）
実行環境: macOS (Apple Silicon M1 Pro) ローカル環境

【重要】Windows固有のコード、パス指定、環境構築手順は一切除外すること。

言語: Python 3.10+

フロントエンド: Streamlit (UI/UX構築)

バックエンド・データ操作: pandas, requests

データベース: Google Sheets (gspread, oauth2client 経由)

データソース: Clash Royale Official API

3. システムアーキテクチャ
システムは以下の2つの独立したモジュールで構成する。

data_pipeline.py (データ収集バッチ)

APIからデータを取得し、JSONをフラットな表形式に変換してスプレッドシートの最下行へ追記する。

重複データの挿入を防ぐため、Match ID等で差分チェックを行うロジックを実装する。

app.py (分析ダッシュボード)

Streamlitを起動し、スプレッドシートから全件データを読み込む。

APIの制限（Quota）を回避するため、必ず @st.cache_data を用いてデータをメモリ上に保持する。

4. 機能要件（フルスペック実装リスト）
A. 基礎分析機能（Basic Analytics）
戦績サマリー: 全体の勝率、直近〇戦の勝率、平均獲得/喪失クラウン数のKPI表示。

デッキ完全一致分析: 自分が使用した8枚のカードセットごとの勝率、使用率のランキング。

時間帯別パフォーマンス: プレイした時間帯（朝、昼、夜、深夜）別の勝率ヒートマップ。

B. 高度な相性分析（Advanced Matchup）
アーキタイプ（型）自動分類: 相手のデッキに含まれるキーカード（例：枯渇、ゴーレム、ホグ）をベースに、デッキタイプを自動判定するロジックの実装。

対面相性マトリクス: 自分のメインデッキ vs 相手のアーキタイプ別の勝率をクロス集計（ピボットテーブル）で可視化。

特定カードのシナジー・天敵分析: 相手のデッキに特定のカード（例：メガナイト）が含まれていた場合の勝率変動の可視化。

C. データサイエンス＆統計機能（Data Science Features）
勝敗の傾向モデリング: pandas や scikit-learn の基礎機能を活用し、「相手の平均カードレベルの差」と「勝率」の相関関係の散布図化。

連勝・連敗トラッカー: 連続試合数と勝率の推移（ティルト状態の分析）。

5. データベース設計 (Google Sheets)
スプレッドシートは以下のカラム構造（スキーマ）を基本とする。

MatchID (一意のID)

Timestamp (試合日時)

GameMode (モード)

MyDeck_Raw (自分のカード8枚の文字列)

OpponentDeck_Raw (相手のカード8枚の文字列)

Result (Win/Loss/Draw)

MyCrowns, OpponentCrowns

MyTrophy, OpponentTrophy
